# Supervisor Review Reports

## What Is This?

This directory contains audit reports generated by the **Supervisor Agent (Opus 4.6)** — an independent reviewer that validates the project's integrity, security, and correctness at each phase gate.

---

---

## 9 Audit Categories

| # | Category | What It Checks |
|---|----------|----------------|
| 1 | **Checkpoint Integrity** | Are checkpoints sequential? Any gaps? Do they match completed tasks? |
| 2 | **Task Table Consistency** | Do task statuses match actual code? Are dependencies correct? |
| 3 | **Model Routing** | Is simple work on Haiku? Complex work on Sonnet? Does it follow CLAUDE.md rules? |
| 4 | **Multi-Tenant Security** | Does every tenant-scoped query include `WHERE galleryId = ?`? Is gallery middleware applied? |
| 5 | **Tax Calculation** | Do formulas match SPEC.md? Is the verification example mathematically correct? |
| 6 | **Prisma Schema** | Does `schema.prisma` match SPEC.md models 1:1? Are indexes, relations, enums correct? |
| 7 | **API Security** | Does every endpoint have auth + role + validation middleware? |
| 8 | **PROJECT_TREE.md** | Is the project tree up to date? Are there stale warnings? |
| 9 | **Risks & Bottlenecks** | What blocks progress? What could go wrong? |

---

## Severity Levels

| Level | Meaning | Action Required |
|---|---|---|
| **KRITIK** | Security hole, data leak risk, blocking issue | Must fix before phase closes |
| **UYARI** | Potential problem, design inconsistency | Should fix before next phase |
| **ONERI** | Improvement, best practice, performance | Nice to have, plan for later |

---

## Verdict Scale

| Verdict | Meaning |
|---|---|
| **ONAY** | All clear. Proceed to next phase. |
| **KOSULLU ONAY** | Proceed, but listed conditions must be met before phase closes. |
| **RED** | Stop. Critical issues must be resolved. Revert to last safe checkpoint if needed. |

## When To Run Supervisor Review

- After every phase completion (before phase gate approval)
- After major schema changes
- After security-sensitive module implementation (auth, calculator, payments)
- On demand via `/supervisor-review` command

---

## How To Request A Review

```bash
# In Claude Code session:
/supervisor-review
```

The supervisor will read all core documents, spawn parallel audit agents, cross-reference findings, and produce a structured report in this directory.
